# Ajay Patil — Portfolio

## Local Development

Install dependencies (one-time):
```bash
npm install
```

Watch for changes during development:
```bash
npm run watch
```

## Building for Production (required before deploying)

```bash
npm run build
```

This generates `dist/output.css` — a minified stylesheet containing **only** the Tailwind classes actually used in the project. The file must be committed to the repo so GitHub Pages can serve it.

## Project Structure

```
├── index.html          # Main HTML
├── app.js              # JavaScript
├── data.json           # Portfolio content (edit this to update the site)
├── src/
│   └── input.css       # Tailwind source (directives + custom component styles)
├── dist/
│   └── output.css      # ← Generated by `npm run build` — commit this file
├── tailwind.config.js  # Tailwind content paths
└── package.json        # Build scripts
```

## Updating Content

All text content (bio, experience, skills, links) lives in `data.json`. Edit that file and refresh — no rebuild needed for content changes. A rebuild is only needed if you add new Tailwind classes to `index.html` or `app.js`.

## GitHub Pages Deployment

1. Run `npm run build`
2. Commit everything including `dist/output.css`
3. Push to your repo
4. Enable GitHub Pages in repo Settings → Pages → Source: `main` branch, root `/`
